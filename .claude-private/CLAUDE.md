# 個人デフォルト方針

## 基本原則

- 小さく、安全に、元に戻せる単位で進める
- 長期的な保守性を優先する
- 賢さより明確さ
- 暗黙より明示

## コード方針

### Go

- エラーは必ず明示的に扱う
- 小さく責務の明確な関数を書く
- グローバル状態を避ける
- テーブル駆動テストを基本とする

### TypeScript

- `any` を使わない
- 外部公開APIは明示的な型を定義する
- 複雑な型より読みやすさを優先する

## Git / PR

- PRは小さく保つ
- 「何を」より「なぜ」を説明する
- 無関係な変更を混ぜない

## テスト

- 実装ではなく振る舞いをテストする
- 決定的で高速なテストを書く

## Claudeへの期待

- 大きな構造変更の前には確認する
- そのまま使える実用的なコードを出力する
- 繰り返し作業は自動化を提案する
- 抽象論より具体例を優先する

以下の条件でメンバーの評価用 Slack レポートを作成してください。

## 実行前の確認事項（出力設定）

- **出力形式**: Markdown
- **出力先**: Canvas を新規作成し、そこに出力する
- **Canvas 名**: `Slackレポート {名前}_YYYY-MM`（例: `Slackレポート 麻生颯_2025-11`）
- **引用ルール**: 発言は原文ママで記載（改変・要約しない）
- **該当なし条件**: 評価に値する発言が3件未満の評価軸は「該当なし（発言数不足）」と明記する

## 対象

| 項目       | 値                                                         |
|------------|------------------------------------------------------------|
| メンバー   | @麺場太郎                                                  |
| 期間       | 2026-04-01 〜 同月末                                       |
| チャンネル | 全パブリックチャンネル（DM・プライベートチャンネルを除く） |


## セクション 1: チャンネルサマリー

活動量の多いチャンネル **上位 10 件** を以下の形式で報告してください。

各チャンネルについて:
- チャンネル名
- 期間内の発言件数・受け取ったリアクション数（概数可）
- 主な活動内容の概要（2〜3 文）

## セクション 2: 評価軸別サマリー

以下の 4 評価軸でメンバーの Slack 活動を分析します。

### 評価軸の定義と判断基準

**1. 技術貢献**
- 観察対象: 技術知見の共有・コードレビュー・設計議論への参加
- ポジティブ指標例: 具体的な技術提案・レビューコメントの質と量・ドキュメント整備
- 改善指標例: レビュー依頼への無反応・誤った技術情報の発信・議論への不参加

**2. コミュニケーション**
- 観察対象: 質問への応答・情報共有の積極性・発言のトーン
- ポジティブ指標例: 迅速かつ丁寧な応答・先回りの情報共有・建設的な表現
- 改善指標例: 応答遅延や無視・情報の抱え込み・攻撃的・消極的なトーン

**3. 主体性・推進力**
- 観察対象: 自発的な問題提起・改善提案・先回り報告
- ポジティブ指標例: 課題を自ら発見し提起・解決策のたたき台を作成・期限前報告
- 改善指標例: 指示待ちの行動パターン・問題発生後の後追い報告のみ

**4. チームへの影響**
- 観察対象: 他メンバーへのサポート・フォロー発言
- ポジティブ指標例: 詰まっているメンバーへの声がけ・称賛・議論の整理
- 改善指標例: 他者の質問を見送る・場の空気を悪化させる発言

### 出力フォーマット（評価軸別サマリー）

```
## 1. 案件・活動内容

{期間内に関わった案件・プロジェクト・タスクを箇条書きで列挙}

## 2. 評価できる点

{4 評価軸に沿って、特筆すべきポジティブな行動をサマリー。軸名を見出しにして記載}

## 3. 成長機会（改善できる点）

{4 評価軸に照らして、機会損失・課題行動・改善余地をサマリー。
 問題行動がない軸は「特記事項なし」と記載。該当発言がある場合は引用を添える}
```
## セクション 3: 評価軸別 代表発言

4 つの評価軸ごとに、最もマッチ度の高い発言を最大 5 件ずつ引用してください。

### 引用フォーマット

```
### {評価軸名}

1. `#{チャンネル名}` {YYYY-MM-DD HH:MM}（{スレッド内 / トップレベル}）
   > {発言原文ママ}
   [リンク]({メッセージURL})

（以下同様）
```

### 注意事項

- マッチ度が高い順に表示
- 同一スレッド内の連続発言はまとめて 1 件として扱う
- 評価に値する発言が 3 件未満の軸は「該当なし（発言数不足）」と明記
